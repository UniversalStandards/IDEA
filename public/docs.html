Universal MCP Orchestration Hub

A universal, intelligent, self-provisioning MCP (Model Context Protocol) orchestration platform that automatically discovers, validates, installs, governs, and operates tools, services, agents, and protocol adapters from public registries, private repositories, enterprise catalogs, and local ecosystems on demand.

Apache-2.0 licensed. Free for commercial use. Contributions welcome via GitHub.

Key features

  • Automatic tool discovery across GitHub, official MCP registries, and private catalogs
  • Zero-touch provisioning — missing tools are installed and configured at runtime
  • Multi-provider AI routing with circuit-breaker and automatic fallback
  • 10-stage trust pipeline with policy enforcement and approval gates
  • Universal protocol support: MCP, JSON-RPC, REST, GraphQL, CLI, WebSocket
  • Structured logging, distributed tracing, and cost monitoring

Quick Start

Get the hub running locally in under two minutes.

Prerequisites

  • Node.js ≥ 20.0.0
  • npm ≥ 10.0.0
  • Git

Clone and run

git clone https://github.com/UniversalStandards/IDEA.git
cd IDEA
npm ci
cp .env.example .env
# Edit .env with your values
npm run dev

The hub starts on http://localhost:3000 by default.

Docker

docker compose up

Docker Compose starts the hub along with a Redis service for future distributed caching. The hub is available at http://localhost:3000.

Installation

From source

npm ci          # Install exact locked dependencies
npm run build   # Compile TypeScript → dist/
npm start       # Run compiled output

Build for production

npm run build
NODE_ENV=production node dist/index.js

Development mode (with hot reload)

npm run dev    # Uses tsx watch — no build step needed

Configuration

All configuration is driven by environment variables, validated at startup with Zod. Copy .env.example to .env and fill in your values.

Core variables

Variable Default Description
PORT 3000 HTTP server port
NODE_ENV development Runtime environment
MCP_TRANSPORT http Transport mode: http | stdio | sse
JWT_SECRET required JWT signing secret (≥ 32 chars)
ENCRYPTION_KEY required 64-char hex key for AES-256-GCM
LOG_LEVEL info Logging verbosity
CORS_ORIGIN * Allowed CORS origins (comma-separated)

See Environment Variables Reference for the full list including rate limiting, webhooks, cost monitoring, and Redis configuration.

Discovery

The discovery engine connects to multiple registries simultaneously and merges the results into a unified capability graph.

Supported registry sources

  • GitHub Registry — scans GitHub repositories tagged with MCP topics
  • Official MCP Registry — the canonical modelcontextprotocol.io registry
  • Enterprise Catalog — internal catalog via HTTP or local file
  • Local Workspace — scans your local filesystem for MCP server configs

Enable enterprise catalog

ENABLE_ENTERPRISE_CATALOG=true
ENTERPRISE_CATALOG_URL=https://catalog.internal.example.com/tools
# or file-based:
ENTERPRISE_CATALOG_PATH=/etc/mcp-hub/catalog.json

Discovery results are cached with a configurable TTL (default 5 minutes). A single failing registry will not block results from other sources.

Provisioning

When a required capability is not installed, the provisioner automatically installs it. The process is: resolve → verify checksum → install → register → activate. 

# Provisioning is automatic. To manually trigger:
POST /admin/tools/install
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "mcp-filesystem",
  "version": "1.2.0",
  "source": "npm"
}

Routing

The provider router selects the best AI provider for each request based on capability, cost, latency, and policy constraints.

Fallback chain

Routes follow: PRIMARY → FALLBACK → LOCAL. A circuit breaker opens after N consecutive failures and routes around the failing provider for a cooldown period.

Configure providers

OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_AI_API_KEY=AIza...

Policy & Trust

Every tool activation passes through a 10-stage trust pipeline before execution:

  1. Discovery metadata inspection
  2. Source repository validation
  3. Package signature verification
  4. Provenance chain check
  5. Allowlist / denylist evaluation
  6. Risk scoring
  7. Environment policy matching
  8. Approval gate (synchronous or async)
  9. Provisioning admission
  10. Runtime monitoring

Policy packs

Drop JSON policy files into the policies/ directory. The engine hot-reloads changes without a restart.

Adapters

Adapters expose external services and tools as MCP capabilities.

MCP Adapter

Connects to the hub via stdio (for Claude Desktop / local CLIs) or SSE (for web-based clients). 

# stdio mode (Claude Desktop)
MCP_TRANSPORT=stdio node dist/index.js

# SSE mode
MCP_TRANSPORT=sse node dist/index.js
# Connect client to: GET http://localhost:3000/mcp/sse

REST Adapter

Wraps any REST API as an MCP tool. Configure via the admin API or a JSON descriptor file.

GraphQL Adapter

Performs schema introspection and maps each query/mutation to an MCP tool. Supports custom auth headers via the credential broker.

CLI Adapter

Executes local CLI tools in a sandboxed subprocess. Shell metacharacters are rejected. Configurable timeout (default 30 s, max 300 s).

Events Adapter

Receives webhook events (POST /adapters/events/webhook) and SSE streams (GET /adapters/events/stream), mapping them to workflow triggers. Events are deduplicated within a configurable window.

API Reference

The hub exposes a REST Admin API on the same port as the MCP server.

Health endpoints

GET /health        # Combined liveness + readiness
GET /health/live   # Always 200 (liveness probe)
GET /health/ready  # 200 when initialized, 503 otherwise

Admin endpoints (require JWT)

GET    /admin/capabilities        # List registered capabilities
DELETE /admin/capabilities/:id    # Deregister a capability
GET    /admin/policies             # List active policy rules
GET    /admin/costs                # Cost summary
GET    /admin/audit                # Recent audit log entries

See docs/api.md for the full API reference with request/response schemas, error codes, and example curl commands.

Deployment

Docker (recommended for production)

docker build -t mcp-hub .
docker run -p 3000:3000 \
  --env-file .env \
  mcp-hub

Docker Compose (with Redis)

docker compose up -d

Health check endpoints for orchestrators

# Kubernetes readiness probe
GET /health/ready

# Kubernetes liveness probe
GET /health/live

For Kubernetes manifests, nginx configuration, and air-gapped deployment instructions, see docs/deployment.md.
